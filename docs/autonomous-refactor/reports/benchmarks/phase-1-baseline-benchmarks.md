## Fixture Notes

All fixtures were generated by the TypeScript benchmark harness. No Rust-written
or mixed TypeScript/Rust input files were used.

| Module | Fixtures |
| --- | --- |
| `analysis_core` | Deterministic synthetic OHLCV arrays: 128, 2,048, and 10,000 bars; direct arithmetic expression strings; invalid formula case. |
| `trading_core` | `MockBroker` account state, Decimal positions/orders, guard pipeline operations, `UnifiedTradingAccount` snapshot input with 25 positions and one pending order. |
| `store_core` | Temp JSONL event logs, temp session JSONL files, generated 1,000-entry session histories, temp news archive JSONL with 5,000 records. |

Per-scenario heap deltas and full fixture text are in
`phase-1-baseline-benchmark-results.json`. Heap deltas were captured with
`process.memoryUsage().heapUsed`; no forced-GC or allocation profiler was used,
so treat them as coarse notes rather than allocation proof.

## Blockers and Gaps

- Rust comparison and TypeScript/Rust binding overhead are not measurable yet:
  the Rust crates and Node bindings do not exist in this repo.
- Rust runtime metadata could not be captured on this machine because `rustc`
  and `cargo` are not installed or not on `PATH`.
- The benchmark harness uses deterministic synthetic fixtures. It does not use
  market/network providers, live broker APIs, or production `data/` files.
- Session disk benchmarks require a temp cwd because `SessionStore` resolves
  its storage root from `process.cwd()` at module evaluation time.
- News archive benchmarks exercise the pure archive functions and
  `NewsCollectorStore` provider path, not the AI tool wrapper factory.

## Follow-up Comparison Rule

Future Rust-backed benchmark reports should run the same harness, add matching
Rust-path scenarios under the same fixture sizes, and compute deltas against
`phase-1-baseline-benchmark-results.json`. Binding overhead should be reported
as a separate scenario instead of being folded into aggregate formula,
indicator, or store operation timings.
